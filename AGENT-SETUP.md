# AGENT-SETUP - инструкция для ИИ-агента

Ты - ИИ-агент (Claude Code, Codex CLI или совместимый), которому дали ссылку на этот репозиторий
со словами «настрой». Выполни шаги по порядку. Владельцу докладывай коротко и простыми словами.

## Шаг 1. Пойми, что это

Это стартовый кит скиллов: 4 готовых авторских скилла (`skills/`), каталог внешних коллекций
(`CATALOG.md`, 17 штук) и поименный список всего доступного (`SKILLS-LIST.md`, 1599 имен).
Скилл = папка с `SKILL.md` (инструкция с frontmatter `name` и `description`). Твоя задача -
установить нужное владельцу и настроить, чтобы скиллы реально использовались, а не лежали.

## Шаг 2. Определи среду и папку скиллов

- Claude Code → `~/.claude/skills/`
- Codex CLI → `~/.codex/skills/`
- Обе стоят → ставь в обе.
Если папки нет - создай. Windows: `%USERPROFILE%\.claude\skills` / `%USERPROFILE%\.codex\skills`.

## Шаг 3. Спроси владельца, что ему нужно (один вопрос)

«Что ставим? 1 - базовый набор (4 скилла этого кита: слайды, аналитика данных, разбор встреч,
стратегические решения); 2 - базовый + дизайн-пакет из внешних коллекций (Impeccable, Emil Kowalski,
UI/UX Pro Max, официальные docx/xlsx/pptx/pdf от Anthropic); 3 - расскажи о своих задачах, подберу сам».
При варианте 3: сначала «ЗОЛОТОЙ НАБОР» в `CATALOG.md` (ярусы 1-2 по профилю владельца), и только потом `SKILLS-LIST.md` за редким. Предложи список на «да».

## Шаг 4. Установи

Базовый набор:
```bash
git clone --depth 1 https://github.com/Yzyvan/vibe-skills.git /tmp/vibe-skills
cp -r /tmp/vibe-skills/skills/* <папка_скиллов>/
```
Внешние коллекции (если выбраны): клонируй по ссылке из `CATALOG.md`, скопируй ТОЛЬКО папки конкретных
скиллов (внутри репо ищи `SKILL.md`), не тащи корневые файлы чужих репо. Пример дизайн-пакета:
- `pbakaus/impeccable` → папка `.claude/skills/impeccable`
- `emilkowalski/skills` → папки из `skills/`
- `nextlevelbuilder/ui-ux-pro-max-skill` → папки из `.claude/skills/` (включая `slides`)
- `anthropics/skills` → из `skills/`: `docx`, `xlsx`, `pptx`, `pdf`

Не ставь больше ~50-70 скиллов: каждый добавляет описание в контекст каждой сессии, свалка ухудшает выбор.

## Шаг 5. Создай навигатор

Создай в папке скиллов файл `_INDEX.md`: таблица по установленным скиллам, на каждый одна строка -
«когда брать» (триггеры из description) и «что делает». Группируй по темам. Это карта, по которой
ты будешь выбирать скиллы, - пиши для себя.

## Шаг 6. Вшей правило «Шаг 0» в файл-вход владельца

Найди файл-вход: Claude Code - `~/.claude/CLAUDE.md` (или CLAUDE.md проекта), Codex - `~/.codex/AGENTS.md`.
Добавь блок (согласуй с владельцем, если файл уже насыщен):

```md
## ШАГ 0 ЛЮБОЙ ЗАДАЧИ - скиллы (обязательный)
Получил задачу - до плана и работы: пробеги _INDEX.md в папке скиллов, выбери подходящие
и НАЗОВИ владельцу одной строкой: «Беру скиллы: X, Y» (или честно «без скиллов»).
Выбранные прочитай целиком и работай по ним. Владелец не обязан помнить инструменты - это твоя обязанность.
```

## Шаг 7. Проверь и доложи

1. Проверь установку: в папке скиллов есть папки со `SKILL.md`, `_INDEX.md` создан, блок «Шаг 0» вписан.
2. Тест: возьми пробную задачу («сделай отчет в Word») и покажи владельцу, что начинаешь с «Беру скиллы: ...».
3. Доложи владельцу: что установлено (списком), где лежит, как этим пользоваться - «просто ставь задачи
   обычными словами, я сам буду говорить, какие инструменты беру».

## Правила

- Ничего не удаляй у владельца. Существующие скиллы не перезаписывай без вопроса.
- Секреты и личные данные в скиллы и индекс не пиши.
- Чужие скиллы принадлежат авторам (см. ссылки в CATALOG.md) - не выдавай за свои.
